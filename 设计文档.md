# 社交媒体自动化发帖框架设计文档

## 一、项目概述

本文档描述了一个通用的社交媒体自动化发帖框架的设计思路和实现方法。该框架采用模块化设计，可以轻松适配不同平台（Quora、Reddit、Twitter/X、Facebook等）。

```python
def check_browser_status(port: int) -> Optional[str]:
    """
    检测指定端口是否有 Chrome 调试模式运行
    
    返回:
        WebSocket 调试地址或 None
    """
    response = requests.get(f"http://127.0.0.1:{port}/json/version")
    return response.json().get("webSocketDebuggerUrl")
```

```python
def quora_posting_logic(page, logger, stop_event, content, image_path=None, **kwargs):
    # 1. 导航到 Quora
    page.goto("https://www.quora.com")
    
    # 2. 点击创建帖子按钮
    page.click('[data-q="CreatePostButton"]')
    
    # 3. 输入内容
    page.fill('[contenteditable="true"]', content)
    
    # 4. 上传图片（如有）
    if image_path:
        page.setInputFiles('input[type="file"]', image_path)
    
    # 5. 发布
    page.click('[data-q="PostButton"]')
    
    return True
```

### 3.5 配置管理层 (config_manager.py)

```python
class ConfigManager:
    CONFIG_FILE = "config.txt"  # JSON 格式
    
    def __init__(self):
        self.config = self._get_defaults()
        self.load()
        
    def _get_defaults(self) -> dict:
        return {
            "chrome_path": r"C:\Program Files\Google\Chrome\Application\chrome.exe",
            "base_profile_path": r"C:\ChromeProfiles",
            "post_count": 1,
        }
        
    def get(self, key: str):
        return self.config.get(key)
        
    def set(self, key: str, value):
        self.config[key] = value
        self.save()
        
    def save(self):
        with open(self.CONFIG_FILE, "w", encoding="utf-8") as f:
            json.dump(self.config, f, indent=4, ensure_ascii=False)
```

### 4.3 日志系统集成

```python
class TextHandler(logging.Handler):
    """将日志输出到 GUI 文本框"""
    
    def __init__(self, text_widget):
        super().__init__()
        self.text_widget = text_widget
        
    def emit(self, record):
        msg = self.format(record)
        self.text_widget.after(0, lambda: self.append_log(msg))
        
    def append_log(self, msg):
        self.text_widget.insert("end", msg + "\n")
        self.text_widget.see("end")
```

### 4.5 任务栏图标设置

```python
def _setup_icon(self):
    icon_path = self._get_resource_path("one.ico")
    if os.path.exists(icon_path):
        self.iconbitmap(icon_path)
        # 设置任务栏图标
        from ctypes import windll
        windll.shell32.SetCurrentProcessExplicitAppUserModelID("app.id.v1.0")

def _get_resource_path(self, relative_path):
    # 支持 PyInstaller 打包后的路径
    if hasattr(sys, '_MEIPASS'):
        return os.path.join(sys._MEIPASS, relative_path)
    return os.path.join(os.path.dirname(__file__), relative_path)
```

## 八、注意事项

### 8.1 浏览器配置
- 每个实例使用独立的用户数据目录
- 端口范围: 9222-9231 (可配置)
- 首次使用需要手动登录账号

### 8.2 打包注意事项
- 使用系统 Python 打包，避免 conda 环境的 tkinter 问题
- 图标文件需要包含在 datas 中
- 隐藏模块需要显式声明

### 8.3 运行环境
- Windows 10/11
- Chrome 浏览器已安装
- Python 3.9+ (开发环境)
